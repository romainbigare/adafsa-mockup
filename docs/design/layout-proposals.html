Wafra dashboard — layout proposals

Three ways to organise the six contract modules for a non-technical government audience. Companion to the written review; wireframes are schematic, not pixel specs.

Context: 25,000 farms · 6 modules (Crop Monitoring 14.9% · Palms & Fruit Trees 31.6% · Structures 21.0% · Irrigation 10.5% · Yield 12.6% · Water 9.4%) · GITEX demo gate on the palms module.

Clean and calm — but it surfaces only land-use and tree distribution; none of the six contract modules exist here yet, and the map is a passive picture rather than the workspace.

Recommendation

Build A2 if the Overview map performs; keep A as the fallback. A2 is A plus a region map on the landing page — the same six module pages, the same reused components, only one extra map instance on Home. It lands the satellite “wow” first, which is exactly what a GITEX room wants, so it’s the stronger demo. The single risk is that one Home map has to draw all 25k boundaries at once; the moment that feels heavy, A2 degrades gracefully to A (drop the map, keep the cards) with zero wasted work. Either way the module switcher moves from seven stacked accordions into the left nav and each module page owns its map outright. If the clock runs out before GITEX, ship B first — it’s mostly deleting duplicate surfaces you already built — and grow it into A/A2 after.
